# edumark
Техническое задание: Полная админ-панель для Edumark
1. Цель проекта
Создать полностью функциональную админ-панель на Filament v3, позволяющую управлять всеми сущностями платформы, с полной интеграцией с фронтендом.

2. Основные разделы админки
2.1. Меню (Menu Management)
Цель: Управление структурой меню с иерархией
Задачи:
Добавить поддержку вложенности (родитель-дочерние пункты)
Добавить типы пунктов: ссылка, выпадающее меню, mega-меню, кнопка
Реализовать сортировку через drag-and-drop
Добавить фильтрацию по устройствам (десктоп/планшет/мобильный)
Включить/выключить видимость пункта
Добавить поддержку иконок и метаданных
2.2. Когорты (Cohorts)
Цель: Управление учебными группами
Задачи:
Создать ресурс для управления когортами
Добавить поля: название, описание, слаг, дата начала
Реализовать выбор цвета/иконки
Добавить связь со студентами (Many-to-Many)
Настроить отображение на фронтенде
2.3. Студенты (Students)
Цель: Управление пользователями и их профилями
Задачи:
Создать профиль студента с полями: специальность, локация, статус
Добавить возможность прикрепления к когортам
Реализовать фильтрацию по когортам в Student Directory
Настроить отображение в разделе "Student Directory" на фронтенде
2.4. Преподаватели (Teachers)
Цель: Управление преподавателями
Задачи:
Создать отдельный тип пользователя "преподаватель"
Добавить поля: специализация, локация, био
Добавить связь с когортами
Настроить отображение в разделе "Ask Your Teacher"
2.5. Ресурсы (Resources)
Цель: Управление учебными материалами
Задачи:
Создать ресурс для управления ресурсами (курсы, слайды, шаблоны)
Добавить типы ресурсов (курсы, слайды, шаблоны, квизы)
Реализовать загрузку файлов
Настроить отображение в разделе каждой когорты
2.6. Задания (Assignments)
Цель: Управление домашними заданиями
Задачи:
Добавить управление заданиями по когортам
Реализовать привязку к календарю
Добавить системы оценок и обратной связи
Настроить отображение в разделе когорты
2.7. Обсуждения (Discussions)
Цель: Управление обсуждениями
Задачи:
Создать категории обсуждений (вопросы, обратная связь, гиды)
Добавить управление топиками и ответами
Реализовать систему лайков и пометки как решения
Настроить отображение в разделе "Community"
2.8. События (Events)
Цель: Управление событиями и вебинарами
Задачи:
Создать ресурс для управления событиями
Добавить поля: дата начала, описание, изображение
Реализовать систему регистрации
Настроить отображение в разделе "Events"
3. Интеграция с фронтендом
3.1. Динамическое меню
Цель: Отображать меню из админки на фронтенде
Задачи:
Создать MenuRepository с кэшированием
Реализовать метод getTree($slug, $location, $device)
Добавить поддержку всех типов пунктов (ссылки, выпадающие меню)
Настроить отображение в layout
3.2. Когорты и контент
Цель: Отображать когорты и их контент
Задачи:
Создать роуты для каждой когорты
Реализовать динамическую загрузку ресурсов, заданий, обсуждений
Настроить отображение в шаблонах когорт
3.3. Адаптация под устройства
Цель: Показывать только подходящие пункты меню
Задачи:
Добавить метод detectDevice() в контроллер
Фильтровать меню по устройству
Настроить адаптивность в Blade-шаблонах
4. Технические требования
4.1. Требования к админке
Использовать Filament v3.3.43
Все сущности должны иметь CRUD-интерфейс
Должна быть поддержка вложенных структур (меню, обсуждения)
Должна быть возможность массового удаления и фильтрации
Все страницы должны иметь поддержку локализации (русский/английский)
4.2. Требования к интеграции
Все данные должны загружаться через Eloquent-модели
Должно быть кэширование данных меню (очистка через админку)
Должна быть поддержка ролей (админ, учитель, студент)
Все функции должны быть защищены правами доступа
5. Детальный план реализации
5.1. Шаг 1: Настройка админки
Убедиться, что все ресурсы правильно настроены
Добавить SubNavigationPosition в MenuResource
Создать все необходимые страницы (List, Create, Edit)
Настроить авторизацию через Authenticate middleware
5.2. Шаг 2: Меню и навигация
Создать модель Menu и MenuItem
Добавить RelationManager для пунктов меню
Реализовать drag-and-drop для сортировки
Настроить отображение в шаблонах
5.3. Шаг 3: Контент-менеджмент
Создать ресурсы для всех сущностей
Добавить формы с необходимыми полями
Настроить отображение в админке
5.4. Шаг 4: Интеграция с фронтендом
Настроить загрузку данных через MenuRepository
Добавить методы в контроллеры для отображения данных
Создать Blade-компоненты для отображения:
Когорт
Ресурсов
Обсуждений
Событий
5.5. Шаг 5: Тестирование
Проверить все функции админки
Проверить отображение данных на фронтенде
Протестировать все сценарии использования

6.1. Кэширование
Добавить очистку кэша меню через админку
Реализовать автоматическое обновление кэша при изменениях
6.2. Поиск
Добавить поисковую строку в админке
Настроить фильтрацию по всем полям
6.3. Экспорт данных
Добавить экспорт данных в CSV/Excel
Реализовать печать отчетов

Полная интеграция с существующей структурой проекта
